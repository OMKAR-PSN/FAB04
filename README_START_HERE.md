## 🔑 Key Insights (TL;DR)

1. **No model to train** — Just a math function (NumPy). No PyTorch, no GPU, no dataset.

2. **Two components** — Variance-based (noise robustness) + Geometry-based (landscape optimization)

3. **Theorem F3 is the key** — Inverting Hessian eigenvalues balances convergence rates. This is why geometry works.

4. **Precompute in __init__** — All expensive math (Hessian eigendecomposition) happens once per seed.

5. **Per-query is fast** — Just two dot products + blend = under 1ms.

6. **Blending is magic** — 70% geometry + 30% variance beats both components separately.

7. **Alpha tuning matters** — Grid search on 5 seeds (hour 12–16) pays off with better final metrics.

## 🚨 Red Flags & Fixes

| Problem | Cause | Fix |
|---|---|---|
| Δ < 0 | Variance direction inverted | Check sign of `q - mu` |
| Spread < 2× | Hessian wrong | Verify H_k formula, eigenvalue range |
| NaN in output | Numerical instability | Add regularization (`H + λI`), clamp eigenvalues |
| Slow __init__ | Full eigendecomposition per query | Move eigh to __init__ only |
| Per-seed regression | Weak attractor match on some seeds | Add confidence gate (fallback to variance) |

## 📚 References & Context

### The Problem (PCAM P-04)
- **Goal:** Improve PCAM retrieval by computing anisotropic precision vectors
- **Input:** K stored patterns (64D each), model parameters (beta, eta, R)
- **Output:** Per-query precision (64D vector, positive, mean=1)
- **Metric:** Δ retrieval accuracy + spread reduction ratio

### The Paper (Theorem F3)
- States: "To equalize local convergence rates, invert Hessian eigenvalues"
- Math: Π = V @ diag(1/√λ) @ V^T
- Implication: High curvature → high precision, flat → low precision

### Why This Approach Works
- Variance alone handles noise robustness (Δ slightly improves)
- Geometry alone handles landscape optimization (spread improves a lot)
- Blending (70/30) gives both benefits simultaneously
- Implementation is fast (precompute once, query in <1ms)

## ❓ Frequently Asked Questions

**Q: Do I need to understand PCAM dynamics?**
A: No. The harness runs the dynamics. You just provide precision. Ignore the dynamics.

**Q: Do I need to train a model?**
A: No. Everything is computed from the stored patterns input at runtime.

**Q: What's the hardest part?**
A: Getting the Hessian eigendecomposition right (P1's job). Everything else is straightforward.

**Q: What if we have < 4 people?**
A: Merge P1 + P2 (one person handles both). P3 can be you or rotate. P4 is essential.

**Q: What if we fail?**
A: Use fallback: set alpha=0.0 (pure variance). Still beats baseline, guaranteed.

**Q: Can we use external libraries?**
A: NumPy only. No scipy, no PyTorch, no TensorFlow (per problem rules).

**Q: How do I know my code is correct?**
A: Quick test should show Δ ≥ 0.04 and spread ≥ 8×. If not, something's wrong.
